# スクリーンショット撮影ガイド

## 📋 Play Store要件

- **必要枚数**: 最低2枚、最大8枚
- **推奨サイズ**: 1080x1920px または 1440x2560px（16:9アスペクト比）
- **フォーマット**: PNG または JPEG
- **推奨枚数**: 4-6枚（多すぎず少なすぎず）

## 🔧 撮影方法

### Android Studioエミュレータを使用

1. **エミュレータ起動**
   ```
   Android Studio > Device Manager > Pixel 6 (または任意の端末)
   解像度: 1080x2340 (推奨)
   ```

2. **アプリ起動とサンプルデータ作成**
   - アプリをエミュレータにインストール
   - 以下のサンプルメモを作成:
     - メモ1: 期限7日（緑色表示）
     - メモ2: 期限12時間（黄色表示）
     - メモ3: 期限2時間（オレンジ表示）

3. **スクリーンショット撮影**
   - エミュレータの右側のツールバー > カメラアイコン
   - または、エミュレータ上で `Ctrl + S`

4. **保存場所**
   - デフォルト: `C:\Users\[ユーザー名]\Desktop`
   - または指定したフォルダ

---

### 実機を使用

1. **開発者オプションを有効化**
   - 設定 > デバイス情報 > ビルド番号を7回タップ

2. **USBデバッグを有効化**
   - 設定 > システム > 開発者向けオプション > USBデバッグ

3. **Android Studioでスクリーンショット**
   - Android Studio > Logcat > カメラアイコン

4. **または、端末のスクリーンショット機能**
   - 音量下ボタン + 電源ボタン
   - ただし、FLAG_SECUREが有効な場合は撮影できないため、
     テスト用に一時的にFLAG_SECUREをコメントアウトする必要あり

## 🎨 撮影のポイント

### DO（推奨）

- ✅ 明るい背景・ダークモード両方で撮影（Play Storeが自動切替）
- ✅ サンプルデータは実用的で分かりやすい内容に
- ✅ 各画面の主要機能が一目で分かるように
- ✅ 統一感のあるスクリーンショット

### DON'T（避ける）

- ❌ 空のメモ一覧（「メモがありません」の表示）
- ❌ 意味のないダミーテキスト（「test」「あああ」など）
- ❌ 個人情報や実在する電話番号・住所
- ❌ 解像度が低い、ぼやけた画像

## ⚠️ トラブルシューティング

### Q: FLAG_SECUREでスクリーンショットが真っ黒になる
**A**: テスト用に一時的にFLAG_SECUREをコメントアウト

MainActivity.kt:
```kotlin
// window.setFlags(
//     WindowManager.LayoutParams.FLAG_SECURE,
//     WindowManager.LayoutParams.FLAG_SECURE
// )
```

**重要**: スクリーンショット撮影後、必ず元に戻すこと！

### Q: エミュレータでメモが表示されない
**A**: アプリを再起動するか、メモを再度作成

### Q: ウィジェットが表示されない
**A**: ホーム画面を長押し > ウィジェット > 一時保存メモを選択
