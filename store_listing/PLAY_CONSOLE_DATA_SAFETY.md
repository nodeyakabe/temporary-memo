# Play Console データセーフティ回答テンプレート

このドキュメントをコピペして使用してください。

## 📋 データセーフティフォーム

### セクション1: データの収集と共有

#### 質問1: アプリでユーザーデータを収集または共有しますか？

**回答**: ❌ いいえ

**説明**:
```
本アプリは、ユーザーの個人情報やアプリのアクティビティデータを一切収集・共有しません。すべてのメモは端末内にのみ保存され、インターネット権限も使用していないため、外部へのデータ送信は不可能です。
```

---

### セクション2: セキュリティプラクティス

#### 質問2-1: データは送信中に暗号化されますか？

**回答**: ⚠️ 該当なし

**理由**:
```
本アプリはインターネット権限（INTERNET permission）を一切使用していないため、データの送信自体が行われません。すべてのデータは端末内にのみ保存されます。
```

---

#### 質問2-2: ユーザーはデータの削除をリクエストできますか？

**回答**: ✅ はい

**説明**:
```
ユーザーはアプリ内でメモを長押しすることで、いつでも自由にメモを削除できます。また、アプリをアンインストールすることで、すべてのデータが完全に削除されます。データはクラウドやサーバーに保存されていないため、端末から削除されれば完全に消去されます。
```

---

#### 質問2-3: データの安全性に関する独立した審査を受けていますか？

**回答**: ❌ いいえ

**説明**:
```
本アプリは独立した第三者機関による審査を受けていません。ただし、ソースコードは以下のセキュリティ対策を実装しています：
- INTERNET権限なし（外部通信不可）
- FLAG_SECURE（スクリーンショット防止）
- allowBackup=false（バックアップ無効化）
- 生体認証によるアクセス制御
```

---

### セクション3: アプリの権限

#### 質問3: アプリが使用する権限とその理由

**USE_BIOMETRIC 権限**:

**使用目的**:
```
アプリのロック機能に使用します。ユーザーが設定で生体認証ロックを有効にした場合、アプリ起動時に指紋認証または顔認証による本人確認を要求します。これにより、第三者による不正なメモの閲覧を防止します。
```

**データ収集との関係**:
```
生体認証データは端末のセキュアストレージに保存されており、アプリ側では一切アクセスしません。認証の成功/失敗の結果のみを受け取ります。
```

## 📝 追加の説明（任意フィールド用）

### アプリのデータ処理について

```
「一時保存メモ」は、ユーザーのプライバシーを最優先に設計されています。

【データの保存場所】
すべてのメモは、端末内のローカルデータベース（Room SQLite）にのみ保存されます。クラウドサービスや外部サーバーへの送信は一切行いません。

【データの暗号化】
メモは暗号化されていません。これは、アプリのシンプルさと「一時的な情報」という性質を重視したためです。重要な機密情報の保存には使用しないでください。

【データの削除】
- ユーザーが手動で削除可能（長押しで削除）
- 設定した期限到達時に自動削除
- アプリアンインストール時に完全削除

【セキュリティ対策】
- インターネット権限なし（通信不可）
- FLAG_SECURE（画面キャプチャ防止）
- バックアップ無効化（機種変更時にデータ残存なし）
- 生体認証ロック（オプション）

【分析・広告】
本アプリは、分析ツール、広告SDK、トラッキングツールを一切使用していません。
```

**回答**: ❌ いいえ

## 🔐 データの削除ポリシー

### ユーザーがデータ削除をリクエストした場合の対応

```
1. **アプリ内での削除**
   - ユーザーはアプリ内でメモを長押しし、削除を選択することでいつでもデータを削除できます。

2. **完全削除**
   - アプリをアンインストールすることで、すべてのデータが端末から完全に削除されます。

3. **サーバー上のデータ**
   - 本アプリはサーバーにデータを保存していないため、サーバー側での削除対応は不要です。

4. **自動削除**
   - 設定した期限が到来すると、メモは自動的に完全削除されます。
```
